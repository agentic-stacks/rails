# Internationalization (i18n)

Rails ships with a full i18n framework. Translations live in YAML files. Views and controllers look up strings by key. Swapping locales changes every translated string without touching application code.

## Setup

Rails creates `config/locales/en.yml` by default. Add one file per locale:

```
config/locales/
├── en.yml
├── fr.yml
└── es.yml
```

Locale files are nested YAML:

```yaml
# config/locales/en.yml
en:
  articles:
    index:
      title: "Articles"
      empty: "No articles yet."
    show:
      published_on: "Published on %{date}"
  common:
    save: "Save"
    cancel: "Cancel"
```

## Translate in Views and Controllers

Use the `t` helper (alias for `I18n.t`):

```ruby
# Full key lookup
t("articles.index.title")        # => "Articles"
t("articles.show.published_on", date: "Jan 1") # => "Published on Jan 1"
```

Use lazy lookup (`.key`) inside a view or controller to infer the scope from the current file path:

```erb
<%# app/views/articles/index.html.erb %>
<h1><%= t(".title") %></h1>   <%# looks up articles.index.title %>
<p><%= t(".empty") %></p>     <%# looks up articles.index.empty %>
```

Lazy lookup in a controller action:

```ruby
# app/controllers/articles_controller.rb
def create
  # ...
  flash[:notice] = t(".created")  # looks up articles.create.created
end
```

## Set the Locale

Set `I18n.locale` for each request. The locale resets between requests.

### Using around_action

```ruby
# app/controllers/application_controller.rb
class ApplicationController < ActionController::Base
  around_action :switch_locale

  private

  def switch_locale(&action)
    locale = extract_locale
    I18n.with_locale(locale, &action)
  end

  def extract_locale
    # Try URL param, fall back to default
    params[:locale].presence_in(I18n.available_locales.map(&:to_s)) ||
      I18n.default_locale
  end
end
```

### From the URL path

```ruby
# config/routes.rb
scope "/:locale", locale: /en|fr|es/ do
  resources :articles
end
```

Access via `/fr/articles`. `params[:locale]` is set automatically.

### From a subdomain

```ruby
def extract_locale
  subdomain = request.subdomain
  subdomain.presence_in(%w[en fr es]) || I18n.default_locale
end
```

### From a cookie or user preference

```ruby
def extract_locale
  locale = current_user&.locale || cookies[:locale]
  locale.presence_in(I18n.available_locales.map(&:to_s)) || I18n.default_locale
end
```

## Configure Available Locales

```ruby
# config/application.rb
config.i18n.available_locales = [:en, :fr, :es]
config.i18n.default_locale = :en
```

## Pluralization

Use the `count` option and `one`/`other` keys (or locale-specific plural forms):

```yaml
# config/locales/en.yml
en:
  inbox:
    messages:
      one: "1 message"
      other: "%{count} messages"
```

```ruby
t("inbox.messages", count: 1)  # => "1 message"
t("inbox.messages", count: 5)  # => "5 messages"
```

Languages with complex pluralization (Russian, Arabic, etc.) require custom plural rules. The `rails-i18n` gem ships these:

```ruby
# Gemfile
gem "rails-i18n"
```

This gem also ships locale files for dates, times, and numbers for dozens of languages.

## Model Translations

Translate model names and attribute labels for use in error messages, form labels, and table headers:

```yaml
# config/locales/en.yml
en:
  activerecord:
    models:
      article:
        one: "Article"
        other: "Articles"
    attributes:
      article:
        title: "Title"
        body: "Body"
        published_at: "Published at"
```

```ruby
Article.model_name.human           # => "Article"
Article.human_attribute_name(:title) # => "Title"
```

Error messages use these translations automatically:

```yaml
en:
  activerecord:
    errors:
      models:
        article:
          attributes:
            title:
              blank: "can't be blank"
```

## Date, Time, and Number Formatting

Use the `l` helper (alias for `I18n.l`) to format locale-aware values:

```ruby
l(Date.today)                   # => "January 1, 2024" (en) or "1 janvier 2024" (fr)
l(Time.now)                     # => "January 1, 2024 12:00"
l(Time.now, format: :short)     # => "1 Jan 12:00"
l(Time.now, format: :long)      # => "January 01, 2024 12:00"
```

Format numbers and currencies:

```ruby
number_to_currency(1234.56)              # => "$1,234.56"
number_to_currency(1234.56, locale: :fr) # => "1 234,56 €"
number_with_delimiter(1234567)           # => "1,234,567"
```

## Fallbacks

Enable fallbacks so that missing translations in one locale fall back to another:

```ruby
# config/application.rb
config.i18n.fallbacks = true
```

With default fallbacks, a missing `fr` key falls back to `en`. Configure explicit fallback chains:

```ruby
config.i18n.fallbacks = { fr: [:fr, :en], "es-MX": [:"es-MX", :es, :en] }
```

## Handle Missing Translations

By default, Rails raises an `I18n::MissingTranslationData` exception in development and renders a `translation missing: en.key` span in production.

Raise in all environments to catch gaps early:

```ruby
# config/initializers/i18n.rb
Rails.application.config.action_view.raise_on_missing_translations = true
```
